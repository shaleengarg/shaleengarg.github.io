check_pte - check if ->page is mapped at the ->pte

page_vma_mapped_walk() found a place where ->page is potentially mapped. check_pte() has to validate this.

->pte may point to empty PTE, swap PTE or PTE pointing to arbitrary page.

If PVMW_MIGRATION flag is set, returns true if ->pte contains migration entry that points to ->page or any subpage in case of THP.

If PVMW_MIGRATION flag is not set, returns true if ->pte points to ->page or any subpage in case of THP.

Otherwise, return false.

◆ page_vma_mapped_walk()

bool page_vma_mapped_walk ( struct page_vma_mapped_walk *  pvmw)

page_vma_mapped_walk - check if ->page is mapped in ->vma at ->address : pointer to struct page_vma_mapped_walk. page, vma, address and flags must be set. pmd, pte and ptl must be NULL.

Returns true if the page is mapped in the vma. ->pmd and ->pte point to relevant page table entries. ->ptl is locked. ->address is adjusted if needed (for PTE-mapped THPs).

If ->pmd is set but ->pte is not, you have found PMD-mapped page (usually THP). For PTE-mapped THP, you should run page_vma_mapped_walk() in a loop to find all PTEs that map the THP.

For HugeTLB pages, ->pte is set to the relevant page table entry regardless of which page table level the page is mapped at. ->pmd is NULL.

Retruns false if there are no more page table entries for the page in the vma. ->ptl is unlocked and ->pte is unmapped.

If you need to stop the walk before page_vma_mapped_walk() returned false, use page_vma_mapped_walk_done(). It will do the housekeeping.
