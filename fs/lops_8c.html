gfs2_log_get_bio - Get cached log bio, or allocate a new one : The superblock : The device block number we want to write to

If there is a cached bio, then if the next block number is sequential with the previous one, return it, otherwise flush the bio to the device. If there is not a cached bio, or we just flushed it, then allocate a new one.

Returns: The bio to use for log writes
