◆ hfsplus_submit_bio()

int hfsplus_submit_bio ( struct super_block *  sb,
sector_t  sector,
void *  buf,
void **  data,
int  op,
int  op_flags 
)

hfsplus_submit_bio - Perform block I/O : super block of volume for I/O : block to read or write, for blocks of HFSPLUS_SECTOR_SIZE bytes : buffer for I/O : output pointer for location of requested data : direction of I/O : request op flags

The unit of I/O is hfsplus_min_io_size(sb), which may be bigger than HFSPLUS_SECTOR_SIZE, and must be sized accordingly. On reads will return a pointer to the start of the requested sector, which may not be the same location as .

If is not aligned to the bdev logical block size it will be rounded down. For writes this means that should contain data that starts at the rounded-down address. As long as the data was read using hfsplus_submit_bio() and the same buffer is used things will work correctly.
