__nilfs_error() - report failure condition on a filesystem

__nilfs_error() sets an ERROR_FS flag on the superblock as well as reporting an error message. This function should be called when NILFS detects incoherences or defects of meta data on disk.

This implements the body of nilfs_error() macro. Normally, nilfs_error() should be used. As for sustainable errors such as a single-shot I/O error, nilfs_msg() should be used instead.

Callers should not add a trailing newline since this will do it.
