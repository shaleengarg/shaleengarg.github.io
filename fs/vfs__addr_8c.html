v9fs_direct_IO - 9P address space operation for direct I/O : target I/O control block

The presence of v9fs_direct_IO() in the address space ops vector allowes open() O_DIRECT flags which would have failed otherwise.

In the non-cached mode, we shunt off direct read and write requests before the VFS gets them, so this method should never be called.

Direct IO is not 'yet' supported in the cached mode. Hence when this routine is called through generic_file_aio_read(), the read/write fails with an error.
