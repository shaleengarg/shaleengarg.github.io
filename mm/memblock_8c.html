__next__mem_range - next function for for_each_free_mem_range() etc. : pointer to u64 loop variable : node selector, NUMA_NO_NODE for all nodes : pick from blocks based on memory attributes : pointer to memblock_type from where the range is taken : pointer to memblock_type which excludes memory from being taken : ptr to phys_addr_t for start address of the range, can be NULL : ptr to phys_addr_t for end address of the range, can be NULL : ptr to int for nid of the range, can be NULL

Find the first area from * which matches , fill the out parameters, and update * for the next iteration. The lower 32bit of * contains index into type_a and the upper 32bit indexes the areas before each region in type_b. For example, if type_b regions look like the following,

0:[0-16), 1:[32-48), 2:[128-130)

The upper 32bit indexes the following regions.

0:[0-0), 1:[16-32), 2:[48-128), 3:[130-MAX)

As both region arrays are sorted, the function advances the two indices in lockstep and returns each intersection.

When allocation direction is bottom-up, the should be greater than the end of the kernel image. Otherwise, it will be trimmed. memblock_virt_alloc_internal - allocate boot memory block : size of memory block to be allocated in bytes : alignment of the region and block's size : the lower bound of the memory region to allocate (phys address) : the upper bound of the memory region to allocate (phys address) : nid of the free area to find, NUMA_NO_NODE for any node

The limit is dropped if it can not be satisfied and the allocation will fall back to memory below . Also, allocation may fall back to any node in the system if the specified node can not hold the requested memory.

The allocation is performed from memory region limited by memblock.current_limit if == BOOTMEM_ALLOC_ACCESSIBLE.

The memory block is aligned on SMP_CACHE_BYTES if == 0.

The phys address of allocated boot memory block is converted to virtual and allocated memory is reset to 0.

In addition, function sets the min_count to 0 using kmemleak_alloc for allocated boot memory block, so that it is never reported as leaks.

RETURNS: Virtual address of allocated memory block on success, NULL on failure.
