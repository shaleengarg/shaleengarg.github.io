Detailed Description

struct tick_sched - sched tick emulation and no idle tick control/stats : hrtimer to schedule the periodic tick in high resolution mode : Store the last tick expiry time when the tick timer is modified for nohz sleeps. This is necessary to resume the tick timer operation in the timeline when the CPU returns from nohz sleep. : Next tick to be fired when in dynticks mode. : Indicator that the idle tick has been stopped : jiffies at the entry to idle for idle time accounting : Total number of idle calls : Number of idle calls, where the sched tick was stopped : Time when the idle call was entered : Time when the idle was interrupted : Time when the idle state was left : Sum of the time slept in idle with sched tick stopped : Sum of the time slept in idle with sched tick stopped, with IO outstanding : Anticipated timer expiration time (in case sched tick is stopped) : Base time clock monotonic for : CPU was the last one doing do_timer before going idle : Tick timer function has run with set
