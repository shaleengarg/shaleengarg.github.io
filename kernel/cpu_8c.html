__cpuhp_setup_state_cpuslocked - Setup the callbacks for an hotplug machine state : The state to setup : If true, the startup function is invoked for cpus where cpu state >= : startup callback function : teardown callback function : State is set up for multiple instances which get added afterwards.

The caller needs to hold cpus read locked while calling this function. Returns: On success: Positive state number if is CPUHP_AP_ONLINE_DYN 0 for all other states On failure: proper (negative) error code
