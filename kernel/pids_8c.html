pids_charge - hierarchically charge the pid count : the pid cgroup state : the number of pids to charge

This function does not follow the pid limit set. It cannot fail and the new pid count may exceed the limit. This is only used for reverting failed attaches, where there is no other way out than violating the limit.

pids_try_charge - hierarchically try to charge the pid count : the pid cgroup state : the number of pids to charge

This function follows the set limit. It will fail if the charge would cause the new value to exceed the hierarchical limit. Returns 0 if the charge succeeded, otherwise -EAGAIN.
