trace_call_bpf - invoke BPF program : tracepoint event : opaque context pointer

kprobe handlers execute BPF programs via this helper. Can be used from static tracepoints in the future.

Return: BPF programs always return an integer which is interpreted by kprobe handler as: 0 - return from kprobe (event is filtered out) 1 - store kprobe event into ring buffer Other values are reserved and currently alias to 1
